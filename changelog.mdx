## Migration Guide

<Callout kind="alert">
When upgrading from v1.0.0 to v1.2.0, review breaking changes related to config files and authentication endpoints.
</Callout>

Follow these steps to migrate smoothly.

<Steps>
  <Step title="Backup Your Data" icon="database">
    Export all projects from the dashboard at `https://dashboard.example.com/export`.
  </Step>
  <Step title="Update Configuration" icon="settings">
    Replace old config with the new format.

    <CodeGroup tabs="JavaScript,Python">
      ```javascript
      // Old config (v1.0)
      module.exports = {
        apiKey: 'YOUR_API_KEY',
        baseUrl: 'https://api.example.com/v1'
      };
      ```
      ```javascript
      // New config (v1.2+)
      module.exports = {
        apiKey: 'YOUR_API_KEY',
        baseUrl: 'https://api.example.com/v2',
        features: { collaboration: true }
      };
      ```
      ```python
      # Old config (v1.0)
      config = {
          "api_key": "YOUR_API_KEY",
          "base_url": "https://api.example.com/v1"
      }
      ```
      ```python
      # New config (v1.2+)
      config = {
          "api_key": "YOUR_API_KEY",
          "base_url": "https://api.example.com/v2",
          "features": {"collaboration": True}
      }
      ```
    </CodeGroup>
  </Step>
  <Step title="Run Migration Script" icon="play">
    Execute the provided script:

    ```bash
    npm run migrate --from=v1.0 --to=v1.2
    ```
  </Step>
  <Step title="Test Integration" icon="check-circle">
    Verify endpoints at `https://api.example.com/v2/docs`.
  </Step>
</Steps>

<Expandable title="Advanced Migration for Custom Integrations" default-open="false">
If you have custom webhooks, update the payload format:

```json
{
  "event": "doc.updated",
  "data": {
    "id": "doc-123",
    "changes": { "title": "Updated Doc" }
  }
}
```

Previously, it used `document.updated` as the event name.
</Expandable>
